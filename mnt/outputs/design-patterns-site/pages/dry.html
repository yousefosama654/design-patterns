What is the DRY Principle?

"Don't Repeat Yourself" is a fundamental software development principle that states every piece of knowledge or logic must have a single, unambiguous representation within a system.

Instead of copying and pasting code across your application, the DRY principle encourages you to abstract common functionality into reusable functions, classes, or modules. This single source of truth makes your code easier to understand, maintain, and modify.

When you violate DRY by duplicating code, you create multiple points of failure. If a bug exists in duplicated code, you need to fix it in multiple places—a recipe for inconsistency and maintenance nightmares.

def calculate_discount(price, user_type): discounts = { "regular": 0.05, "vip": 0.1, "employee": 0.2 } discount_rate = discounts.get(user_type.lower(), 0) discount = price * discount_rate total = price - discount print(f"{user_type.capitalize()} customer: Price after discount is ${total:.2f}") return total price = 100 calculate_discount(price, "regular") calculate_discount(price, "vip") calculate_discount(price, "employee")

Key Takeaways

🐛

Fewer Bugs

Fix logic once, and it's fixed everywhere. Duplicated code means duplicated bugs—DRY eliminates this problem.

🔧

Easier Maintenance

Update features in one place. Changes propagate automatically across your entire codebase, saving time and reducing errors.

✨

Cleaner Code

Less duplication means less code to read and understand. Your codebase becomes more elegant and professional.
